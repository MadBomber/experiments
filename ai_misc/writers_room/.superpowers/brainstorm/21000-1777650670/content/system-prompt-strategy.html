Section 4: System Prompt Strategy

How Actor + Character + Casting + Scene layers compose into each robot_lab agent's system prompt

Prompt Composition Layers

Layer 1 — Actor consistent across all their characters

actor.name, actor.style_notes

"Michael J. Fox brings nervous energy, comedic timing, and everyman relatability to every role."

Layer 2 — Character persistent across all projects

character.personality, character.voice_pattern, character.current_arc

"Marty McFly: optimistic, quick-witted, loyal to friends. Speaks in 80s slang. Currently learning that recklessness has consequences."

Layer 3 — Casting scoped to this project

casting.arc_notes, casting.objectives, casting.relationship_notes

"In Back to the Future, Marty must get his parents to fall in love or he'll never be born. He's increasingly desperate."

Layer 4 — Scene scoped to this scene

scene.context, scene.beat_structure, scene_character.scene_objectives, cast list

"Scene 1 — The Clock Tower. Marty's objective: convince Doc the time machine works before lightning strikes."

User Prompt — Runtime changes each turn

Last N transcript lines + current beat + "What does [character] say?"

- 1-3 sentences of dialog only
- No stage directions or name prefix

DirectorService — Building the Agent

# app/services/director_service.rb
def build_agent(character, scene)
  casting = character.castings
    .find_by(project: scene.project)
  actor = casting.actor

  RobotLab.build(
    name: character.name,
    system_prompt: SystemPromptBuilder
      .new(character:, casting:,
           actor:, scene:).build,
    model: character.model
            || actor.preferred_model,
    provider: character.provider
              || actor.preferred_provider
  )
end

Response Decision Logic (adapted from current experiment)

Always respond

Character's name is mentioned in the most recent line of dialog

Respond if appropriate

Last speaker wasn't this character AND they haven't spoken in the last 3 lines more than once

10% random interjection

Preserves natural conversation flow — characters can jump in unexpectedly
