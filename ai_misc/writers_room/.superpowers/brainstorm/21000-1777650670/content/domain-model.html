Revised Domain Model

Actor ↔ Project through Characters — confirm this is right before we design the full app

Entity Relationships

Actor has_many :characters
Actor has_many :projects, through: :characters

Project has_many :characters
Project has_many :actors, through: :characters
Project has_many :scenes

Character belongs_to :actor # Clint in this role
Character belongs_to :project # in this film
Character # + LLM agent state

Scene belongs_to :project
Scene has_many :scene_characters # cast list

SceneRun belongs_to :scene
SceneRun has_many :transcript_lines

The Clint Eastwood Analogy

Actor: Clint Eastwood
Consistent LLM persona, model preference, token budget
Project: A Fistful of Dollars
Character: Joe (The Man with No Name)
Project: The Good, the Bad and the Ugly
Character: Blondie
Project: A Comedy Play (same project, two roles)
Character: Sheriff Dawson + Narrator

Character = the pivot between Actor and Project

Character is not just a join table — it's the LLM agent model. It carries: the role's personality, voice pattern, arc, objectives (the character definition) plus model, provider, data, token counts (the robot_lab agent state). One Actor can have multiple Characters in the same Project.
