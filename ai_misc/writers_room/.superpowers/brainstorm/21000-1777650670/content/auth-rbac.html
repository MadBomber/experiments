Producer is the superuser. A user can hold multiple roles simultaneously.

passwordless Auth Flow

1. User enters email at /auth/sign_in
2. passwordless emails a magic link (token)
3. User clicks link → session established
4. current_user available in all controllers
5. Pundit policies check roles on every action

Pundit Policy Pattern (example)

# app/policies/scene_policy.rb
class ScenePolicy < ApplicationPolicy
  def run?
    user.has_role?(:director)
      || user.has_role?(:producer)
  end
  def create?
    user.has_role?(:writer)
      || user.has_role?(:producer)
  end
end

Gem Stack Addition

passwordless

Magic-link email authentication. No passwords stored. Provides session management and the sign-in flow out of the box.

pundit

Policy objects — one file per resource. Clean fit with Phlex since there's no view layer magic. authorize called in every controller action.

User#has_role?

Simple helper method on User. No gem needed — roles are just a has_many :through and roles.exists?(name:).
