Scene Run Flow (Real-Time)

1. User clicks Run Scene → POST /scene_runs
2. SceneRun created (status: queued), SceneRunJob enqueued
3. Turbo Stream redirect → live transcript page
4. SceneRunJob starts: DirectorService builds robot_lab agents per Character
5. Each dialog line → saved as TranscriptLine → Turbo Stream broadcast to browser
6. Scene ends → SceneRun status: completed, token counts updated on Characters

robot_lab Integration

# DirectorService builds one robot per character
robot = RobotLab.build(
  name: character.name,
  system_prompt: character.system_prompt,
  model: character.model
          || actor.preferred_model,
  provider: character.provider
            || actor.preferred_provider
)
# Streaming callback broadcasts each line
robot.run(prompt) do |chunk|
  Turbo::StreamsChannel.broadcast_...
end

YAML Import / Export

Import: Upload or paste YAML → YamlImportService upserts Actors, Characters, Projects, Scenes, Castings.

Export: Any Project → YamlExportService serializes to the same YAML format the old experiment used, preserving compatibility.

Primary UI Navigation

Actors

Create, edit actors. View their characters and projects.

Characters

Define roles, personality, voice. View token usage history.

Projects

Manage productions. Assign castings. Browse scenes.

Scenes

Write scene details, beats, cast list. Run and view transcripts.

Import / Export

Round-trip YAML for all project data.
