Domain Model v3

Scene → Characters → Actor via Castings

Associations

Actor
has_many :castings
has_many :characters, through: :castings
has_many :projects, through: :castings

Character # IS the LLM agent
has_many :castings
has_many :actors, through: :castings
has_many :projects, through: :castings
has_many :scene_characters
has_many :scenes, through: :scene_characters

Casting # Actor+Character+Project
belongs_to :actor
belongs_to :character
belongs_to :project
# arc_notes, objectives (project-specific)
# unique on [character_id, project_id]

Project
has_many :castings
has_many :characters, through: :castings
has_many :actors, through: :castings
has_many :scenes

Scene # unique to a Project
belongs_to :project
has_many :scene_characters
has_many :characters, through: :scene_characters
has_many :scene_runs

SceneCharacter # cast list for a scene
belongs_to :scene
belongs_to :character

SceneRun
belongs_to :scene
has_many :transcript_lines

Finding the Actor in a Scene

# Who plays Marty in BTTF2's clock scene?
scene.project
  .castings
  .find_by(character: marty)
  .actor
# => Michael J. Fox

The Recast Scenario

Character: Batman
Casting: Batman in Batman (1989)
→ Actor: Michael Keaton
Casting: Batman in Batman Begins (2005)
→ Actor: Christian Bale

Same character, different actors, different projects — all naturally supported.

Key Schema Note

LLM agent state lives on Character

Marty McFly's conversation history, model, provider, and token counts persist across all projects he appears in — he remembers BTTF1 when he enters BTTF2. The Casting holds only the project-specific framing (arc notes, objectives).
