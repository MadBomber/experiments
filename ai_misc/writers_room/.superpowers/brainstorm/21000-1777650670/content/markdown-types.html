Markdown File Types

Seven file types · YAML frontmatter for structured fields · body varies by type

Every file: type: in frontmatter identifies the kind. Body sections use ## Heading prose — no rigid schema, just convention per type.

Actor

---
type: actor
name: Michael J. Fox
preferred_model: ...
preferred_provider: ...
---
## Description
## Style Notes
## Notable Works
## Performance Notes

Character

---
type: character
name: Marty McFly
archetype: Everyman Hero
model: ...
provider: ...
---
## Personality
## Voice Pattern
## Character Arc
## Motivation
## Internal Conflict
## Physical Description
## Mannerisms

Project

---

## Logline

Story — the narrative treatment

---
type: story
project: back_to_the_future
act_structure: three_act
---
## Story Arc
## Act One
## Act Two
## Act Three
## Plot Points
## Conflict Escalation
## Resolution

The full prose narrative treatment — a separate file from the project metadata. One project can have multiple story drafts.

Arc — character arc within a project

---
type: arc
character: marty_mcfly
project: back_to_the_future
actor: michael_j_fox
---
## Arc Description
## Arc Start State
## Arc End Goal
## Key Turning Points
## Current Position

Also encodes the Casting (actor ↔ character ↔ project) and the CharacterArc in one file.

Research Material — can attach to project, character, or scene

---
type: research
subject: 1955 Hill Valley Culture
category: world_building
project: back_to_the_future
character: # optional
scene: # optional
sources: ["...", "..."]
---
## Summary
## Key Facts
## World Building Notes
## Accuracy Requirements
## Sources

Research files are freeform — the body is whatever the writer needs to capture. The frontmatter anchors them to the right project, character, or scene so they surface in the right context within the app.

Import Routing — one dispatcher, seven handlers

# MarkdownImportService reads type: from frontmatter and dispatches
case frontmatter["type"]
when "actor" then ActorImporter.call(frontmatter, sections)
when "character" then CharacterImporter.call(frontmatter, sections)
when "project" then ProjectImporter.call(frontmatter, sections)
when "story" then StoryImporter.call(frontmatter, sections)
when "arc" then ArcImporter.call(frontmatter, sections) # also creates Casting
when "scene" then SceneImporter.call(frontmatter, sections)
when "research" then ResearchImporter.call(frontmatter, sections)
end
