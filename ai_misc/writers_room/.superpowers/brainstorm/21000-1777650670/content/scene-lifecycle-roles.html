Scene Status State Machine

draft Writer is working on the scene

Visible to: Producer, Writer only

Writer clicks "Submit for Review"
ready_for_review Awaiting Director approval

Visible to: Producer, Writer, Director

Director rejects → back to draft
Director approves → released
released Approved — visible to cast

Visible to: All roles. Actor-users see only scenes their character is cast in

Scene schema addition:
status: enum (draft | ready_for_review | released)
submitted_at: datetime # when writer submitted
released_at: datetime # when director approved
released_by: user_id # which director approved

User ↔ Actor Link

An Actor record can optionally have a linked User account. When a user holds the actor role their account is associated with one Actor record — giving them a personalised view of only the released scenes they appear in.

Actor
has_one :user

User
belongs_to :actor, optional: true
# only set when role includes :actor

A Producer assigns the actor role to a user and links them to their Actor record. One user = one Actor record. An Actor record can exist without a linked User account.

Actor Role — What They See

My Characters — all characters this actor plays across all projects
My Scenes — released scenes where at least one of their characters appears, grouped by project
Scene Detail — the scene script (beat structure, objectives) + the latest released transcript
