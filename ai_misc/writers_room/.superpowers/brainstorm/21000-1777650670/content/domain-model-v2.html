Revised Domain Model v2

Character is independent — the same Character can appear across many Projects

Entity Relationships

Actor has_many :characters
Actor has_many :projects, through: :characters (→ castings)

Character belongs_to :actor
Character has_many :castings
Character has_many :projects, through: :castings
Character # LLM agent state (persistent across projects)

Casting belongs_to :character
Casting belongs_to :project
Casting # project-specific: objectives, arc_notes

Project has_many :castings
Project has_many :characters, through: :castings
Project has_many :actors, through: :characters
Project has_many :scenes

Scene belongs_to :project
Scene has_many :scene_castings # who's in this scene
Scene has_many :castings, through: :scene_castings

SceneRun belongs_to :scene
SceneRun has_many :transcript_lines

The Back to the Future Analogy

Actor: Michael J. Fox
Consistent comedic timing, nervous energy, everyman style
Character: Marty McFly
Personality, voice pattern, arc — one persistent LLM agent state that carries memory across films
Casting: Marty in BTTF 1
arc_notes: "discovers time travel, gets home"
Casting: Marty in BTTF 2
arc_notes: "protects future family" — Marty remembers BTTF1!
Casting: Marty in BTTF 3
arc_notes: "rescues Doc in the Old West"

Casting = the pivot between Character and Project

Casting holds the project-specific context for a character (their objectives and arc notes within THIS project) while the Character's LLM agent state persists across all projects — Marty McFly at the start of BTTF2 remembers everything from BTTF1. One Actor can have multiple Characters (plays), and the same Character can appear in multiple Projects (sequels).
