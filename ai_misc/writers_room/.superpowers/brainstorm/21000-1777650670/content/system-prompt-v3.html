System Prompt Strategy v3

Arcs and plot points belong to the Project. Character holds only timeless definition attributes.

Revised Prompt Layers

Layer 1 — Actor

actor.name, actor.style_notes

"Michael J. Fox brings nervous energy and comedic timing."

Layer 2 — Character timeless — unchanged across all projects

character.personality, character.voice_pattern

"Marty McFly: optimistic, quick-witted. Speaks in 80s slang."

✗ current_arc removed — arc belongs to the project, not the character

Layer 3 — Project the production's narrative context

project.description, project.genre, project.setting, project.tone, project.story_arc, project.plot_points

"BTTF — Marty must get his parents together or he won't exist. Plot points: meets 1955 Doc, Enchantment Under the Sea, lightning strike."

+ character_arcs join — per-character arc notes within this project (see schema right)

Layer 4 — Scene

scene.context, scene.beat_structure, scene_character.scene_objectives, cast list

"Clock Tower — convince Doc to proceed before lightning."

User Prompt — Runtime

Last N transcript lines + "What does [character] say?"

Two Separate Project Joins

Casting: links an Actor to a Character in a Project
→ answers "who is playing this character?"

CharacterArc: links a Character's narrative arc to a Project
→ answers "what is this character going through in this production?"

Does this separation look right? Casting = pure actor↔character↔project link. CharacterArc = narrative notes for a character within a project (arc, objectives, relationships). Project = overall story arc and plot points. If so, the full design is complete.
